# GitHub Pagesデプロイ手順 ## 1. GitHubアカウントの準備 GitHub Pagesを利用するには、GitHubのアカウントが必要です。まだアカウントをお持ちでない場合は、[GitHub](https://github.com/)にアクセスして新規登録してください。 ## 2. 新しいリポジトリの作成 1. GitHubにログインした状態で、右上のプロフィールアイコンの隣にある「+」ボタンをクリックし、「New repository」を選択します。 2. リポジトリ名を入力します。例えば「sosina-landing-page」などがわかりやすいでしょう。 - GitHub Pagesで公開する場合、`{ユーザー名}.github.io` という名前にすると、独自のURLでサイトが公開されます。例：`sosina.github.io` - 通常のリポジトリ名の場合は、`{ユーザー名}.github.io/{リポジトリ名}` という形式のURLになります。 3. 「Public」を選択します（Private リポジトリでも GitHub Pages は利用可能ですが、設定が異なります）。 4. 「Initialize this repository with a README」にチェックを入れておくと便利です。 5. 「Create repository」ボタンをクリックして、リポジトリを作成します。 ## 3. ファイルのアップロード ### Webブラウザから直接アップロードする場合 1. 作成したリポジトリページで「Add file」→「Upload files」をクリックします。 2. 作成した以下のファイルをドラッグ＆ドロップまたは選択してアップロードします： - `index.html` - `styles.css` - `script.js` - ロゴ画像ファイル（ある場合） 3. 「Commit changes」ボタンをクリックして、ファイルをコミットします。 ### Git CLIを使用する場合 Git CLIに慣れている方は、以下のコマンドでも操作できます： ```bash # リポジトリをクローン git clone https://github.com/{ユーザー名}/{リポジトリ名}.git cd {リポジトリ名} # ファイルを追加 # 必要なファイルをこのディレクトリにコピー # 変更をステージング git add . # コミット git commit -m "初回コミット：ランディングページファイル" # GitHubにプッシュ git push origin main ``` ## 4. GitHub Pagesの設定 1. リポジトリページで「Settings」タブをクリックします。 2. 左側のメニューから「Pages」を選択します。 3. 「Source」セクションで、ブランチを「main」に、フォルダを「/(root)」に設定します。 4. 「Save」ボタンをクリックします。 5. 設定が保存されると、GitHubは自動的にサイトの構築を開始します。 6. しばらく待つと（通常は数分以内）、「Your site is published at https://{ユーザー名}.github.io/{リポジトリ名}/」というメッセージが表示されます。 ## 5. サイトの確認と更新 1. 表示されたURLにアクセスして、サイトが正しく公開されているか確認します。 2. 変更を加えたい場合は、ローカルでファイルを編集し、再度コミット・プッシュするか、GitHub上で直接編集できます。 3. 変更をプッシュすると、GitHub Pagesは自動的に再構築され、数分後に変更が反映されます。 ## 6. カスタムドメインの設定（オプション） 独自ドメインを使用したい場合は、以下の手順で設定できます： 1. リポジトリの「Settings」→「Pages」画面で、「Custom domain」セクションに独自ドメイン（例：`www.sosina.co.jp`）を入力します。 2. 「Save」ボタンをクリックします。 3. DNSサービスプロバイダーで、以下のいずれかの設定を行います： - Apexドメイン（例：`sosina.co.jp`）の場合： - 4つのAレコードを追加：`185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153` - サブドメイン（例：`www.sosina.co.jp`）の場合： - CNAMEレコードを追加：`{ユーザー名}.github.io` ## 7. HTTPS の有効化（推奨） GitHub Pagesでは、HTTPSを有効にすることを推奨します： 1. リポジトリの「Settings」→「Pages」画面で「Enforce HTTPS」にチェックを入れます。 2. これにより、サイトへのアクセスは自動的にHTTPSにリダイレクトされ、セキュリティが向上します。 ## 注意点 - GitHub Pagesは静的ホスティングサービスのため、サーバーサイドの処理（PHP, Rubyなど）は実行できません。 - 公開リポジトリの場合、サイトのソースコードは一般公開されます。 - GitHub Pagesには容量制限（1GBまで）があります。 以上の手順で、株式会社SosinaのランディングページをGitHub Pagesで公開できます。
